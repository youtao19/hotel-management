## 🎯 重构目标

将账单表的 `room_fee`（房费）和 `deposit`（押金）两个独立字段合并为统一的 `change_price` 字段，并通过 `change_type` 字段区分金额类型。

### 核心变更

**变更前**：
```sql
room_fee NUMERIC(10,2)        -- 房间价格
deposit NUMERIC(10,2)          -- 押金
change_price NUMERIC(10,2)    -- 改价金额
change_type TEXT               -- 改价类型
```

**变更后**：
```sql
change_price NUMERIC(10,2)    -- 统一金额字段（收入为正，支出为负）
change_type TEXT               -- 类型：房费、收押、退押、补收、退款
```

### 表结构（最终）

```sql
CREATE TABLE bills (
    bill_id SERIAL PRIMARY KEY,
    order_id VARCHAR(50) NOT NULL,
    room_number VARCHAR(10) NOT NULL,
    guest_name VARCHAR(50),
    change_price NUMERIC(10,2) DEFAULT 0,  -- 统一金额字段
    change_type TEXT,                       -- 类型：房费、收押、退押、补收、退款
    pay_way VARCHAR(50) NOT NULL,
    create_time TIMESTAMP NOT NULL,
    remarks TEXT,
    stay_type TEXT,
    stay_date DATE,
    FOREIGN KEY (order_id) REFERENCES orders(order_id)
);
```

### 1. 后端兼容层
`getOrderBillDetails()` 函数聚合新格式数据并返回旧格式：
```javascript
// 按日期分组，聚合房费和押金
const dateMap = new Map();
for (const row of result.rows) {
    if (row.change_type === '房费') {
        dateData.room_fee += Number(row.change_price || 0);
    } else if (row.change_type === '收押') {
        dateData.deposit += Number(row.change_price || 0);

```

### 2. 入住流程优化
办理入住时创建独立的账单记录：
```javascript

// 仅第一天创建押金记录

```

### 3. 交接班统计重构
根据 `change_type` 分类统计：
```javascript
if (change_type === '房费' || change_type === '收押') {
    hotelIncome[pay_way] += amount;  // 收入
} else if (change_type === '退押') {
    hotelDeposit[pay_way] += Math.abs(amount);  // 支出
```

## ⚠️ 注意事项

### 1. 不兼容的变更
- 旧字段 `room_fee` 和 `deposit` 已永久删除
- 如需回滚，必须从数据库备份恢复

### 2. 新账单创建规范
所有新创建的账单必须指定 `change_type`：
- `房费` - 房间费用（正数）
- `收押` - 收取押金（正数）
- `退押` - 退还押金（负数）
- `补收` - 补收款项（正数）
- `退款` - 退款（负数）

### 3. 查询注意事项
- 查询订单账单详情时，建议使用 `billModule.getOrderBillDetails()`
- 该函数已处理聚合逻辑，返回包含 `room_fee` 和 `deposit` 的兼容格式
- 直接 SQL 查询需要自行聚合数据
