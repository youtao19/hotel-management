# 测试文件更新说明

## 文件路径
`/Users/peach/develop/hotel-management/backend/tests/checkYesterdayHandover.test.js`

## API 变更说明

### 旧API行为
- 前端传入"当前日期" (例如: 2024-01-16)
- 后端计算前一天 (2024-01-15)
- 查询前一天的交接记录

### 新API行为
- 前端直接传入"要查询的日期" (例如: 2024-01-15)  
- 后端直接查询该日期的交接记录
- 不做任何日期计算

## 测试文件需要修改的地方

### 1. 文件头部注释
```javascript
// 旧
测试接口：GET /api/handover/check-yesterday
测试场景：检查昨日交接记录

// 新
测试接口：GET /api/handover/check-yesterday
API行为：前端传入要查询的日期，后端直接查询该日期
测试场景：检查指定日期的交接记录
```

### 2. 测试用例中的日期引用

**所有测试都需要修改为：**
```javascript
// 旧方式
const testDate = '2024-01-15';
// ... 插入数据到 testDate
const queryDate = '2024-01-16';  // 传入"次日"
const res = await request(app).get('/api/handover/check-yesterday').query({ date: queryDate });
expect(res.body.data.date).toBe('2024-01-15');  // 期望返回"前一天"

// 新方式  
const testDate = '2024-01-15';
// ... 插入数据到 testDate
const queryDate = testDate;  // 直接传入testDate
const res = await request(app).get('/api/handover/check-yesterday').query({ date: queryDate });
expect(res.body.data.date).toBe(testDate);  // 期望返回testDate本身
```

### 3. 响应消息的断言

```javascript
// 旧
expect(res.body.message).toBe('昨日已完成交接');
expect(res.body.message).toBe('昨日交接记录不完整');
expect(res.body.message).toBe('昨日无交接记录');

// 新
expect(res.body.message).toBe('已完成交接');
expect(res.body.message).toBe('交接记录不完整');
expect(res.body.message).toBe('无交接记录');
```
