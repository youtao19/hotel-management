# 删除身份证号和手机号改为可选 - 修改说明

**日期**: 2025-10-13  
**版本**: v1.2  
**状态**: ✅ 已完成

---

## 📋 修改概述

简化订单创建流程：
1. **删除身份证号字段** - 不再强制要求身份证号
2. **手机号改为可选** - 允许无联系方式的订单

---

## 🎯 修改原因

### 1. 业务需求变化
- 前台快速办理入住时，不需要强制登记身份证号
- 部分客人可能不愿意提供手机号
- 简化录入流程，提高工作效率

### 2. 实际使用场景
- **现场入住**: 前台快速办理，只需姓名和房间信息
- **电话预订**: 客人可能只留下姓名，不提供其他信息
- **团队预订**: 只需要团队负责人信息

---

## 🔧 修改内容

### 1. 数据库表结构

#### 文件: `backend/database/postgreDB/tables/order.js`

**删除字段**:
```sql
-- 删除
id_number VARCHAR(30) NOT NULL, -- 证件号码

-- phone 字段从 NOT NULL 改为可空
phone VARCHAR(20), -- 客人电话（非必填）
```

#### 数据库迁移
```sql
-- 1. 删除 id_number 列
ALTER TABLE orders DROP COLUMN IF EXISTS id_number;

-- 2. 将 phone 列改为可空
ALTER TABLE orders ALTER COLUMN phone DROP NOT NULL;
```

### 2. 后端代码修改

#### 文件: `backend/modules/orderModule.js`

**修改点 1: 移除必填字段验证**
```javascript
// 旧代码（已删除）
const requiredFields = [
  { field: 'guest_name', name: '客人姓名' },
  { field: 'phone', name: '联系电话' },  // ❌ 删除
  { field: 'id_number', name: '身份证号' }, // ❌ 删除
  ...
];
```

**修改点 2: 创建订单 SQL**
```javascript
// 旧代码
INSERT INTO orders (
  order_id, id_source, order_source, guest_name, phone, id_number, ...
) VALUES ($1, $2, $3, $4, $5, $6, ...)

// 新代码
INSERT INTO orders (
  order_id, id_source, order_source, guest_name, phone, ...
) VALUES ($1, $2, $3, $4, $5, ...)
```

**修改点 3: 更新订单字段**
```javascript
// 移除 id_number
const updateableFields = ['guest_name', 'phone', 'room_type', ...];
```

### 3. 前端代码修改

#### 文件: `frontend/src/pages/CreateOrder.vue`

**修改点 1: 删除身份证号输入框**


**修改点 4: 删除验证函数**
```javascript

```

## ⚠️ 影响分析

### 1. 现有数据
- **现有订单**: 不受影响，phone 和 id_number 字段保留（直到被删除）
- **历史数据**: id_number 列删除后，历史数据丢失（如需保留请先备份）

### 2. 相关功能
#### ✅ 不受影响
- 订单查询
- 订单更新
- 入住办理
- 退房结算
- 报表统计

#### ⚠️ 需要注意
- **导入/导出**: 如果有数据导入导出功能，需要更新字段映射
- **第三方集成**: OTA 平台等可能提供身份证号，需要处理或忽略
