# 退押金统计问题修复

## 问题描述

在交接班流程的步骤4"确认交接数据"表格中，"客房退押"和"休息退押"列显示的金额始终为0，无法正确统计退押金数据。

## 问题原因

### 数据库中的账单类型

后端数据库中的账单类型 (`change_type`) 有以下值：
- `'房费'` - 房费收入
- `'收押'` - 收取押金
- `'订单账单'` - 订单相关账单
- `'退押'` - **退还押金**（问题关键）
- `'退款'` - 退款
- `'补收'` - 补收费用

### 前端代码的判断逻辑

在 `CheckData.vue` 的 `calculateSummaryData()` 函数中，原代码判断退押金时使用的是：

```javascript
if (changeType === '退押金') {
  // 统计退押金...
}
```

但实际上数据库返回的类型值是 `'退押'` 而不是 `'退押金'`，导致条件永远不匹配，退押金统计始终为0。

## 解决方案

修改 `CheckData.vue` 中的账单类型判断逻辑，兼容数据库实际使用的类型值：

### 1. 客房数据统计修复

```javascript
// 修改前
if (changeType === '房费') {
  summaryDataObject.value.hotelIncome[normalizedPayWay] += amount
} else if (changeType === '押金') {
  summaryDataObject.value.hotelDeposit[normalizedPayWay] += amount
} else if (changeType === '退押金') {  // ❌ 永远不匹配
  summaryDataObject.value.hotelRefundDeposit[normalizedPayWay] += Math.abs(amount)
}

// 修改后
if (changeType === '房费') {
  summaryDataObject.value.hotelIncome[normalizedPayWay] += amount
} else if (changeType === '收押' || changeType === '押金') {  // ✅ 兼容两种写法
  summaryDataObject.value.hotelDeposit[normalizedPayWay] += amount
} else if (changeType === '退押' || changeType === '退押金') {  // ✅ 兼容两种写法
  summaryDataObject.value.hotelRefundDeposit[normalizedPayWay] += Math.abs(amount)
  console.log(`💰 [退押统计] 客房退押: ${bill.orderNo}, 类型: ${changeType}, 金额: ${amount}, 支付方式: ${normalizedPayWay}`)
}
```

### 2. 休息房数据统计修复

同样的修复应用到休息房数据统计：

```javascript
// 修改后
if (changeType === '房费') {
  summaryDataObject.value.restIncome[normalizedPayWay] += amount
} else if (changeType === '收押' || changeType === '押金') {  // ✅ 兼容两种写法
  summaryDataObject.value.restDeposit[normalizedPayWay] += amount
} else if (changeType === '退押' || changeType === '退押金') {  // ✅ 兼容两种写法
  summaryDataObject.value.restRefundDeposit[normalizedPayWay] += Math.abs(amount)
  console.log(`💰 [退押统计] 休息房退押: ${bill.orderNo}, 类型: ${changeType}, 金额: ${amount}, 支付方式: ${normalizedPayWay}`)
}
```

### 3. 调试日志增强

添加了详细的控制台日志，便于追踪退押金统计：

```javascript
console.log(`💰 [退押统计] 客房退押: ${bill.orderNo}, 类型: ${changeType}, 金额: ${amount}, 支付方式: ${normalizedPayWay}`)
```

## 修复后的效果

修复后，系统能够正确识别和统计退押金数据：

1. **正确识别退押类型**：能匹配数据库中的 `'退押'` 类型
2. **正确计算金额**：使用 `Math.abs(amount)` 取绝对值，因为退押金在数据库中是负数
3. **按支付方式分类**：正确将退押金按现金、微信、微邮付、其他分类统计
4. **在表格中显示**：步骤4的交接表格中正确显示退押金额

## 数据流程

```
后端 bills 表
  ↓ change_type = '退押'
  ↓ change_price = -200 (负数)
  
前端 CheckData.vue
  ↓ calculateSummaryData()
  ↓ 匹配 changeType === '退押' ✅
  ↓ Math.abs(-200) = 200
  ↓ hotelRefundDeposit['现金'] += 200
  
步骤4 ShiftHandoverPaymentTable
  ↓ 显示：客房退押 = 200 ✅
```
