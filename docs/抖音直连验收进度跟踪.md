## 1. 状态说明

- `未开始`：还未进入开发。
- `进行中`：已开始开发、联调或自测。
- `已实现待验收`：代码已完成，但还没在抖音验收后台通过。
- `已完成`：代码与验收都已完成。
- `阻塞`：依赖外部条件、权限、测试账号或上游接口，暂时无法继续。

## 6. 最近一次同步结论

### 2026-03-24
- 已确认当前仓库可支撑部分交易验收，但无法覆盖整套直连验收。
- 已确认当前最大的缺口是“商品侧 + 价量态 + 可订检查 + 履约同步”。
- 已新增本进度文档，后续每次推进抖音直连相关工作时必须同步更新。
- 已开始并完成基础版“可订检查 SPI”接入，支持正式验签、mock 联调、库存不足和价格不一致分支。
- 已补充“确认接单/拒单”能力，支持手动回传 `confirmResult=1/2`，并把本地订单状态回写为 `confirmed/rejected/failed`。
- 已补充基础版“价量态主链路”，支持手动预览 ARI、推送房量房态、推送房价、触发抖音拉取通知。
- 已补充基础版“入住/离店状态同步”，支持手动按本地 `orderId` 推送入住和离店，并记录最近一次履约推送结果。
- 已补充基础版“创建物理房型”，支持按本地 `localRoomType + poiId` 手动创建抖音物理房型，并自动回查详情写入本地映射。
- 已补充基础版“创建日历房商品”，支持按本地 `localRoomType + roomId + mode` 手动创建抖音售卖房型，并把返回的 `rate_plan_id` 合并回物理房型记录。
- 已补充基础版“商品上下架”，支持按 `roomId + ratePlanId + active` 手动切换抖音商品状态，并同步回写本地 `rate_plan_list.active`。
- 已补充基础版“物理房型上下架”，支持按 `roomId + active` 手动切换抖音物理房型状态，并同步回写本地 `raw_payload.active`。
