# 登出功能修复说明

## 问题描述
系统在用户登出后不会清除 Redis 中的 session 数据，导致 session 残留。

## 问题原因
虽然代码中调用了 `req.session.destroy()`，但由于使用的 `connect-redis` v5.2.0 版本，`destroy()` 方法可能存在不稳定的情况，无法确保 100% 从 Redis 中删除 session。

## 修复方案

### 1. 改进 `authentication.js` 中的 logout 函数

**修改文件**: `/backend/modules/authentication.js`

**主要改进**:
- 引入 `redisDB` 模块，以便直接操作 Redis
- 在 `req.session.destroy()` 之后，显式调用 `redis.del()` 删除 session
- 添加详细的日志记录，便于追踪登出过程
- 使用双重保障机制：先调用 `session.destroy()`，再显式删除 Redis key

**关键代码**:
```javascript
const logout = function (callback) {
    const req = this;
    if (!callback) {
        return new Promise(async (resolve, reject) => {
            try {
                const sessionID = req.sessionID;
                const redis = redisDB.getClient();
                
                delete req.session.account;
                delete req.session.authenticated;
                
                // 销毁 session
                req.session.destroy(async function (err) {
                    if (err) {
                        console.error(`登出失败: session ${sessionID} 销毁出错:`, err);
                    } else {
                        console.log(`session.destroy() 完成: ${sessionID}`);
                    }
                    
                    // 显式从 Redis 中删除 session，确保清理完成
                    try {
                        if (redis && sessionID) {
                            const redisKey = 'sess:' + sessionID;
                            const deleted = await redis.del(redisKey);
                            if (deleted > 0) {
                                console.log(`登出成功: 已从 Redis 中显式删除 session ${sessionID}`);
                            } else {
                                console.log(`Redis 中未找到 session ${sessionID}，可能已被 destroy() 删除`);
                            }
                        }
                    } catch (redisErr) {
                        console.error(`从 Redis 删除 session 时出错:`, redisErr);
                    }
                    
                    return resolve(null);
                });
            } catch (e) {
                console.error('登出异常:', e);
                return reject(e);
            }
        });
    } else {
        // 回调版本（相同逻辑）
        ...
    }
};
```

### 2. 优化 `userRoute.js` 中的登出路由

**修改文件**: `/backend/routes/userRoute.js`

**主要改进**:
- 将登出路由移到 `ensureAuthenticated` 中间件之前，这样即使 session 过期也能登出
- 添加详细的日志记录
- 删除了文件末尾重复的登出路由定义

**关键代码**:
```javascript
// 登出路由不需要认证检查，放在认证中间件之前
router.get("/logout", async (req, res, next) => {
    try {
        console.log('收到登出请求, sessionID:', req.sessionID);
        
        if (req.session && req.logout) {
            await req.logout();
            console.log('登出操作完成');
        } else {
            console.log('session 或 logout 方法不存在，跳过登出');
        }
        
        return res.status(200).json({ message: '登出成功' });
    } catch (e) {
        console.error('登出操作失败:', e);
        return res.status(200).json({ message: '登出完成' });
    }
});

// 其他路由需要认证
router.use(authentication.ensureAuthenticated);
```

## 修复效果

### 修复前
- 用户登出后，Redis 中的 session 数据仍然存在
- 可能导致安全问题和内存泄漏

### 修复后
- 用户登出时，会同时调用 `session.destroy()` 和 `redis.del()`
- 确保 Redis 中的 session 被完全清除
- 添加了详细的日志记录，便于监控和调试
- 即使 session 已过期，用户仍然可以成功登出

## 测试建议

1. **手动测试**:
   ```bash
   # 1. 启动后端服务
   npm run dev
   
   # 2. 登录系统
   # 3. 查看 Redis 中的 session (使用 redis-cli)
   redis-cli
   KEYS sess:*
   
   # 4. 登出系统
   # 5. 再次查看 Redis 中的 session
   KEYS sess:*
   # 应该看到刚才的 session 已被删除
   ```

2. **查看日志**:
   登出时应该看到类似以下的日志输出：
   ```
   收到登出请求, sessionID: xxx
   session.destroy() 完成: xxx
   登出成功: 已从 Redis 中显式删除 session xxx
   登出操作完成
   ```

## 技术细节

- **Redis Key 格式**: `sess:{sessionID}`（前缀在 `app.js` 中配置）
- **connect-redis 版本**: v5.2.0
- **session 过期时间**: 2周（在 `appSettings/setup.js` 中配置）

## 后续优化建议

1. 考虑升级 `connect-redis` 到最新版本（目前 v7.x），新版本对 session 销毁的支持更好
2. 可以添加定期清理过期 session 的定时任务
3. 考虑添加自动化测试，验证登出功能的正确性
