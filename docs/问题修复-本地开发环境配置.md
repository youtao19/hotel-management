# 问题修复总结 - 本地开发环境配置

## 📅 时间
2025年10月11日

## 🐛 问题描述

用户在本地运行 `npm start` 时遇到以下错误：
```
Error: getaddrinfo ENOTFOUND backend
```

前端无法连接到后端 API。

## 🔍 根本原因

**问题**：前端的 proxy 配置默认使用了 Docker 服务名 `backend:3000`

```javascript
// frontend/quasar.config.js
devServer: {
  proxy: {
    '/api': {
      target: process.env.VITE_API_BASE || 'http://backend:3000', // ❌ 默认值错误
    }
  }
}
```

**环境分析**：
- **Docker 环境**：容器间通过服务名通信（`backend`、`postgres`、`redis`）
- **本地开发**：服务运行在宿主机，使用 `localhost`
- **冲突点**：配置默认值假定在 Docker 环境中运行

## ✅ 解决方案

### 1. 修复前端 Proxy 配置

**修改文件**：`frontend/quasar.config.js`

**修改前**：
```javascript
devServer: {
  proxy: {
    '/api': {
      target: process.env.VITE_API_BASE || 'http://backend:3000',
      changeOrigin: true
    }
  }
}
```

**修改后**：
```javascript
devServer: {
  proxy: {
    '/api': {
      // 默认使用 localhost（本地开发），Docker 通过环境变量覆盖
      target: process.env.VITE_API_BASE || 'http://localhost:3000',
      changeOrigin: true
    }
  }
}
```

**效果**：
- ✅ 本地开发：自动使用 `http://localhost:3000`
- ✅ Docker 环境：通过 `VITE_API_BASE=http://backend:3000` 覆盖

### 2. 修复注册成功判断逻辑

**修改文件**：`frontend/src/pages/Register.vue`

**问题**：响应拦截器已返回 `data`，但代码检查 `response.data`

**修改前**：
```javascript
if (response.data) {  // ❌ 错误：多套了一层
  // 注册成功逻辑
}
```

**修改后**：
```javascript
if (response && response.id) {  // ✅ 正确：直接检查 response
  // 注册成功逻辑
}
```

### 3. 修复 Redis API 调用（node-redis v4+）

**修改文件**：`backend/routes/authRoute.js`

**问题**：使用了旧版本的 Redis API

**修改内容**：

#### 3.1 修复 scan 方法

**修改前**：
```javascript
const [nextCursor, scannedKeys] = await redis.scan(cursor, "MATCH", "emailVerification*");
```

**修改后**：
```javascript
const result = await redis.scan(cursor, { MATCH: "emailVerification*" });
const nextCursor = result.cursor;
const scannedKeys = result.keys;
```

#### 3.2 修复 set 方法

**修改前**：
```javascript
await redis.set("emailVerification" + code, req.body.email, "EX", 600);
```

**修改后**：
```javascript
await redis.set("emailVerification" + code, req.body.email, { EX: 600 });
```

#### 3.3 修复 cursor 类型

**修改前**：
```javascript
let cursor = "0";
cursor = nextCursor;  // nextCursor 是数字
```

**修改后**：
```javascript
let cursor = 0;  // 使用数字类型
cursor = result.cursor;  // 保持数字类型
```

## 📝 经验教训

### 1. 环境差异处理

**问题**：
- Docker 环境和本地开发环境有本质区别
- 不能假定所有开发者都使用同一种环境

**解决方案**：
- 配置默认值应该针对最常见的使用场景（本地开发）
- 通过环境变量支持其他场景（Docker、生产环境）
- 提供清晰的文档说明不同环境的配置方法

### 2. 第三方库 API 变更

**问题**：
- node-redis 从 v3 升级到 v4 后，API 发生了重大变化
- 旧代码使用了不兼容的 API

**解决方案**：
- 查看库的迁移指南
- 更新所有相关调用
- 统一使用新 API 的最佳实践

### 3. 响应拦截器的影响

**问题**：
- axios 响应拦截器返回 `response.data`
- 但代码中又检查 `response.data.xxx`，导致多套一层

**解决方案**：
- 明确响应拦截器的行为
- 在所有使用 API 的地方保持一致的数据访问方式
- 添加注释说明拦截器的作用

## 🔄 后续改进建议

### 1. 环境检测

在 `quasar.config.js` 中添加自动环境检测：

```javascript
const isDocker = process.env.DOCKER_ENV === 'true';
const apiTarget = isDocker ? 'http://backend:3000' : 'http://localhost:3000';

devServer: {
  proxy: {
    '/api': {
      target: process.env.VITE_API_BASE || apiTarget,
      changeOrigin: true
    }
  }
}
```

### 2. 统一配置管理

创建配置管理模块：

```javascript
// config/index.js
const config = {
  development: {
    apiUrl: 'http://localhost:3000',
    dbHost: 'localhost',
    redisHost: 'localhost'
  },
  docker: {
    apiUrl: 'http://backend:3000',
    dbHost: 'postgres',
    redisHost: 'redis'
  },
  production: {
    apiUrl: process.env.API_URL,
    dbHost: process.env.DB_HOST,
    redisHost: process.env.REDIS_HOST
  }
};

export default config[process.env.NODE_ENV || 'development'];
```

### 3. 健康检查改进

添加启动前的环境检查：

```javascript
// scripts/check-environment.js
const checks = {
  postgres: () => checkConnection('localhost', 5432),
  redis: () => checkConnection('localhost', 6379),
  docker: () => checkDockerRunning()
};

async function preflightCheck() {
  for (const [service, check] of Object.entries(checks)) {
    const status = await check();
    if (!status) {
      console.error(`❌ ${service} is not running`);
      process.exit(1);
    }
  }
  console.log('✅ All services are ready');
}
```

### 4. 开发脚本改进

在 `package.json` 中添加：

```json
{
  "scripts": {
    "dev:local": "NODE_ENV=development npm start",
    "dev:docker": "docker compose up -d",
    "check": "node scripts/check-environment.js",
    "start": "npm run check && npm run dev:local"
  }
}
```
