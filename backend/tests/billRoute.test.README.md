## 业务规则验证

### 1. 账单过滤规则
```javascript
// 房费、收押、订单账单：按 stay_date 过滤
if (change_type IN ('房费', '收押', '订单账单')) {
  WHERE stay_date = '2025-10-07'
}

// 退押、退款、补收：按 create_time 过滤
if (change_type IN ('退押', '退款', '补收')) {
  WHERE DATE(create_time) = '2025-10-07'
}
```

### 2. 数据分组规则
- 客房账单：`stay_type = '客房'`
- 休息房账单：`stay_type = '休息房'`

### 3. 原始数据返回
- 同一订单的多条账单分开返回
- 每条账单保留独立的 bill_id
- 不做任何金额聚合

## 注意事项

1. **数据库连接**：测试需要连接到测试数据库
2. **数据清理**：测试前后会自动清理 `TEST_` 开头的测试数据
3. **事务处理**：使用事务确保测试数据的一致性
4. **日期格式**：使用 `YYYY-MM-DD` 格式
5. **超时设置**：测试超时时间设置为 30 秒
6. **连接池管理**：测试结束后自动关闭数据库连接池，避免 Jest 超时
7. **外键约束**：测试自动创建必要的房型和房间，避免外键约束错误

## 故障排查

### 测试失败常见原因

1. **数据库连接失败**
   - 检查数据库配置
   - 确认数据库服务运行中

2. **测试数据冲突**
   - 手动清理遗留的测试数据
   ```sql
   DELETE FROM bills WHERE order_id LIKE 'TEST_%';
   DELETE FROM orders WHERE order_id LIKE 'TEST_%';
   ```

3. **字段缺失**
   - 确认 orders 表有 stay_type 字段
   - 确认 bills 表结构完整

4. **时区问题**
   - 测试使用本地时间
   - 确保服务器和数据库时区一致

5. **Jest 超时问题**
   - 确保在 `afterAll` 中关闭数据库连接池
   - 检查是否有未关闭的数据库连接
   - 设置合适的超时时间 `jest.setTimeout(30000)`

6. **外键约束错误**
   - 确保在测试前创建必要的房型和房间
   - 使用 `ON CONFLICT DO NOTHING` 避免重复插入
   - 测试结束后按正确顺序清理数据（先删除账单、订单，再删除房间、房型）
